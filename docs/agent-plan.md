# Agent 助手重构开发计划

## 1. 目标

将当前助手从单次请求模式重构为状态化任务模式，支持流式进度、全量数据存储、上下文预算、待确认恢复、一次性写入确认、用户反馈、个性化记忆、自我优化和动态引导词。

## 2. 阶段一：协议与数据结构

- 定义 `AgentRun`、`AgentRunEvent`、`AgentContextRequest`、`AgentContextObservation`、`AgentAction`、`AgentActionPreview`、`AgentExecutionResult`。
- 定义任务阶段：理解中、读取账本、推理中、校验中、等待确认、执行收尾、完成、失败。
- 定义一次性操作 ID 和重复执行拦截规则。
- 定义 `AgentFeedbackEvent`、`AgentUserMemory`、`AgentSuggestion`。
- 定义旧 `/api/agent/chat` 与新任务流接口的兼容边界。
- 验收：协议文档、后端 Schema、Android DTO 能一一对应。

## 3. 阶段二：后端 Harness 基座

- 建立 FastAPI 任务流接口、上下文继续接口和确认继续接口。
- 建立状态图节点：输入校验、意图路由、上下文规划、上下文中断、模型推理、操作生成、结构校验、确认中断、执行反馈、最终回复、异常恢复。
- 建立工具注册表，所有工具使用明确输入输出 Schema。
- 建立 SSE 事件输出，保证事件顺序稳定。
- 建立数据存储，保存账本、对话、偏好、反馈、任务阶段、模型输入输出和评估结果。
- 建立上下文预算，控制每次模型调用的数据量、耗时和成本。
- 验收：后端单元测试覆盖状态图路由、上下文请求、确认中断、恢复、结构校验、资源预算和异常恢复。

## 4. 阶段三：Android 本地任务层

- 新增本地 `agent_messages`、`agent_runs`、`agent_pending_actions` 存储。
- 新增本地 `agent_feedback_events` 和 `agent_local_profile` 存储。
- 实现 `AgentRunStore`，保存最近消息、任务阶段和待确认操作。
- 实现 `AgentEventReducer`，把后端事件转换为页面状态。
- 实现 `AgentContextProvider` 新观察结果格式。
- 实现 `AgentActionExecutor`，确认后只通过现有 UseCase 写入。
- 实现 `AgentFeedbackCollector`，采集点赞、点踩、重新生成和取消确认事件。
- 验收：断网、退出重进、待确认恢复、重复确认拦截和反馈采集均可验证。

## 5. 阶段四：助手页面改造

- 增加任务阶段提示。
- 增加流式回复和进度展示。
- 增加上下文范围提示。
- 增加可恢复确认卡片。
- 增加候选账目选择后的继续处理。
- 增加点赞、点踩和重新生成小图标。
- 增加动态引导词区域。
- 保持页面轻量，不展示内部节点、工具名、提示词和原始 JSON。
- 验收：助手页在空会话、处理中、等待确认、成功、失败、长对话下表现稳定。

## 6. 阶段五：写入与查账主路径

- 打通新增支出、收入、转账的确认写入。
- 打通查账和统计问答。
- 打通改账、删账和候选选择。
- 打通定时记账创建、修改和停用。
- 打通用户取消、执行失败和继续补充。
- 验收：写入类操作必须确认后执行；查询类问题不写入；失败不产生半完成数据。

## 7. 阶段六：记忆、自我优化与动态引导

- 建立用户记忆生成任务，提取常用分类、账户、表达习惯和回答偏好。
- 建立反馈评分规则，综合点赞、点踩、重新生成、追问、确认和失败。
- 建立动态引导词生成任务，根据日期、账本、对话和偏好生成 2 到 4 条短建议。
- 建立优化分析任务，输出提示词、工具路由、上下文策略和错误恢复的改进候选。
- 建立发布门槛，低风险个人记忆可自动更新，全局策略变更需评估或人工确认。
- 验收：用户记忆可更新，动态引导词不固定，反馈能影响后续回答。

## 8. 阶段七：安全、日志与限流

- 后端允许保存完整账本、完整对话、模型输入输出、偏好和反馈。
- 禁止保存用户自定义 API Key 和设备原始标识。
- 接入设备限流、请求大小限制和模型输出长度限制。
- 接入长期存储体积、后台优化频率和单次上下文预算限制。
- 增加禁止建议拦截，避免投资、借贷、保险、税务、法律和医疗建议。
- 验收：安全测试不保存禁止数据，资源限制生效。

## 9. 阶段八：兼容、清理与发布准备

- 保留旧接口兼容路径，明确后续移除时间点。
- 迁移旧 `agent_conversation_snapshot` 到结构化本地存储。
- 清理旧单次请求 UI 状态和无用 DTO。
- 更新 `docs/code-map.md` 和部署文档中与实际路径相关的内容。
- 完成 Android 与后端最小发布自测。
- 验收：离线记账不受影响，助手主路径通过，代码库没有临时文件和死代码。
