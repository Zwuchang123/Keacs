# Keacs 迭代计划

## 填写要求

- `状态` 只允许填写：`未开始 / 进行中 / 已完成 / 已阻塞 / 已取消`。
- `完成情况说明` 必须记录当前结果、关键变更或阻塞原因；已完成提交的需补充提交号。
- 若实际实现偏离原计划，必须先更新本文件，再继续推进开发。

## 拆分原则
- 先完成工程、主题、导航和测试入口，再进入业务开发。
- 分类、账户和本地数据是记账前置条件，先做稳定。
- 首页、账单和统计依赖核心记账结果，放在余额规则完成后。
- 导入导出依赖数据结构稳定，放在主要业务闭环之后。
- 每轮迭代都要能单独验收，并按当轮可用能力完成构建、静态检查、自动测试、离线检查和必要的人工验证。
