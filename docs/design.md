# Keacs UI 规范

## 1. 规范范围

本文件只保留长期稳定的 UI 规范，用于约束视觉风格、布局方式、组件表现、状态反馈、无障碍和 Compose 落地方式。

## 2. 设计原则

Keacs 是完全离线、无需账号的个人记账 App，界面风格必须稳定保持以下特征：

| 项目 | 规范 |
| --- | --- |
| 产品气质 | 简单、轻量、清爽、可信赖 |
| 使用感受 | 打开即能理解、单手易操作、低学习成本 |
| 视觉方向 | 浅色背景、蓝色主操作、白色卡片、圆形图标、信息层级清楚 |
| 内容原则 | 少说明、少装饰、少打扰，优先展示核心信息 |
| 禁止出现 | 登录、云同步、会员、营销横幅、复杂理财、装饰性大插画 |

参考 `docs/UI-refercence.png`：整体使用柔和浅色手机界面，蓝色只用于主按钮、选中态和关键图表；分类使用彩色圆形图标；页面保持安静，不堆说明文字。

## 3. 布局规范

- 使用 `KeacsScaffold`，页面背景统一为 `Background`。
- 内容左右边距统一为 `16dp`。
- 顶部避开状态栏，底部避开导航栏和键盘。
- 主内容优先采用列表、分组和自然留白，不把所有内容都包成卡片。
- 禁止嵌套卡片，避免层级过深。
- 标题栏遵循统一结构：返回在左、标题居中、主操作在右。
- 模块之间保持稳定节奏，默认模块间距为 `16dp`。
- 长内容优先纵向滚动，禁止无意义横向滚动。
- 底部内容不得被系统栏、浮层或键盘遮挡。

## 4. 视觉规范

颜色策略为克制型：大面积使用浅色中性背景，蓝色只用于主操作和选中态。设计源使用 OKLCH，Android 可落地为近似色值。

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| Primary | `#3F82F6` | 主按钮、选中态、重点图表线 |
| Primary Light | `#EAF2FF` | 选中背景、浅蓝提示 |
| Background | `#F6F8FC` | 页面背景 |
| Surface | `#FCFDFF` | 卡片、输入框、列表 |
| Surface Subtle | `#F1F4F8` | 搜索框、分段控件、键盘块 |
| Text Primary | `#1F2937` | 主文字、金额 |
| Text Secondary | `#6B7280` | 副标题、说明值 |
| Text Tertiary | `#A0A7B3` | 占位、弱信息 |
| Border | `#E6EBF2` | 分割线、输入框边框 |
| Income | `#35C785` | 收入、正向金额 |
| Expense | `#FF5A5F` | 支出、负向金额 |
| Warning | `#FFB020` | 风险提示、确认提醒 |
| Error | `#E5484D` | 删除、失败、错误提示 |

分类色只用于图标底色和图表，不用于大面积背景。

| 场景 | 字号 | 字重 | 行高 |
| --- | --- | --- | --- |
| 页面标题 | `18sp` | `600` | `26sp` |
| 模块标题 | `15sp` | `600` | `22sp` |
| 正文 | `14sp` | `400` | `20sp` |
| 辅助文字 | `12sp` | `400` | `17sp` |
| 大金额 | `30sp` | `700` | `38sp` |
| 卡片金额 | `24sp` | `700` | `32sp` |
| 导航文字 | `11sp` | `500` | `14sp` |

- 使用 Android 系统默认字体。
- 金额使用等宽数字。
- 禁止负字距。
- 文字层级优先靠字号、字重、颜色和留白区分，不靠堆叠装饰。

| 项目 | 规范 |
| --- | --- |
| 基础单位 | `4dp` |
| 页面左右边距 | `16dp` |
| 模块间距 | `16dp` |
| 卡片内边距 | `16dp` |
| 列表项最小高度 | `64dp` |
| 卡片圆角 | `12dp` |
| 输入框圆角 | `8dp` |
| 主按钮圆角 | `10dp` |
| 图标容器 | `40dp` 圆形 |
| 主要悬浮按钮 | `56dp` 圆形 |
| 最小点击区域 | `48dp x 48dp` |
| 阴影 | 只用轻阴影，不用重阴影 |

## 5. 组件规范

| 组件 | 规范 |
| --- | --- |
| `KeacsScaffold` | 负责页面外壳、背景、状态栏和底部安全区 |
| `KeacsTopBar` | 负责返回、标题和右侧操作位 |
| `KeacsBottomBar` | 负责底部导航容器与选中态表现 |
| `CategoryIcon` | 使用圆形底色和白色线性图标 |
| `SegmentedTabs` | 使用浅底容器和蓝色选中态 |
| `AmountInput` | 强调金额输入，使用大号等宽数字 |
| `FormFieldRow` | 统一表单行样式、间距和点击反馈 |
| `RecordListItem` | 统一列表项的信息密度和金额对齐方式 |
| `EmptyState` | 使用小图标、标题和短文案，不默认放主操作 |
| `ConfirmDialog` | 使用标题、正文、取消、确认四段式结构 |
| `KeacsSnackbar` | 用于轻量结果反馈，不承载复杂说明 |

通用约束：

- 所有可点击组件必须有默认、按下、焦点、禁用、加载、错误状态。
- 同类操作只使用一种样式，不混用多个强调级别。
- 图标统一使用 Compose Material Icons Rounded。
- 常规图标尺寸为 `24dp`，图标容器内图标可用 `20dp`。
- 禁止使用 Emoji、混用多套图标、无含义装饰图标。
- 图标优先表达功能和内容，不靠造型堆砌差异。

## 6. 状态与反馈

| 状态 | 规范 |
| --- | --- |
| 加载 | 使用骨架或小进度，不遮挡已有内容 |
| 空状态 | 显示图标、标题和短提示，避免堆长说明 |
| 保存失败 | 使用 Snackbar 或就近错误提示，并保留用户输入 |
| 删除确认 | 必须二次确认 |
| 风险操作 | 使用明确弹窗说明结果和影响 |
| 文件或权限异常 | 明确说明无法读取、保存或执行原因 |
| 大字体 | 金额不截断，按钮不挤压，列表不重叠 |

表单规则：

- 输入错误要就近显示，不只在页面顶部提示。
- 保存中禁止重复提交。
- 失败后保留已输入内容，禁止静默清空。
- 危险或不可逆操作必须确认。
- 弹窗本身不滚动；若选项较多，只允许弹窗内部选项区滚动。

## 7. 动效与无障碍

| 场景 | 时长 | 动效 |
| --- | --- | --- |
| 页面进入 | `220ms` | 淡入加轻微位移 |
| 页面退出 | `180ms` | 淡出加轻微位移 |
| 列表增删 | `180ms` | 使用 Compose LazyList 项位移动画 |
| 分段切换 | `160ms` | 背景滑动、文字变色 |
| 按钮按下 | `80ms` | 涟漪或透明度反馈 |
| 图表加载 | `300ms` | 线条或透明度渐入 |
| 弹窗 | `180ms` | 淡入、轻微缩放 |

无障碍规则：

- 点击区域不小于 `48dp x 48dp`。
- 文字对比度不低于 `4.5:1`。
- 图标按钮必须提供 `contentDescription`。
- 颜色不能作为唯一信息表达方式，金额和状态必须配合文字、符号或数值。
- TalkBack 顺序必须与视觉顺序一致。
- 必须支持系统大字体。
- 键盘弹出时，正在编辑的输入框不能被遮挡。

## 8. Compose 落地约束

- 使用 Jetpack Compose + Material 3。
- 主题入口统一为 `KeacsTheme`。
- 颜色、字体、圆角、间距从主题或设计 token 获取。
- 业务页面不得硬编码颜色和尺寸。
- 页面状态使用不可变 `UiState`。
- 列表使用稳定 `key`。
- 导航使用类型安全路由。
- ViewModel 发出一次性导航事件。
- 预览至少覆盖默认、大字体、空状态、错误状态。

## 9. UI 验收要点

- 风格保持统一：顶部栏、底部栏、卡片、列表项、表单、弹窗语言一致。
- 信息层级清楚：重点信息一眼可见，弱信息不抢主视觉。
- 页面保持安静：没有营销噪音、无关装饰和过量解释文字。
- 交互反馈完整：加载、空态、错误态、禁用态、确认态齐全。
- 布局稳定：没有文字重叠、按钮溢出、横向滚动和遮挡问题。
- 适配正常：支持 `360dp` 宽度、小屏设备和系统大字体。
