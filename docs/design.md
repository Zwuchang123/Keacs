# Keacs 前端设计规范

## 1. 设计目标

Keacs 是完全离线、无需账号的个人记账 App。界面要让用户打开后马上知道：还剩多少钱、最近花了什么、从哪里记一笔。

| 项目   | 规范                          |
| ---- | --------------------------- |
| 产品气质 | 简单、轻量、清爽、可信赖                |
| 使用场景 | 日常消费后，单手快速补一笔账              |
| 视觉方向 | 浅色背景、蓝色主操作、白色卡片、圆形分类图标、底部导航 |
| 首屏任务 | 看概览、看最近记录、快速记收入或支出          |
| 信息原则 | 高频功能放前面，低频功能收进“我的”          |
| 禁止出现 | 登录、云同步、会员、营销横幅、复杂理财、装饰性大插画  |

参考 `docs/UI-refercence.png`：整体使用柔和浅色手机界面，蓝色只用于主按钮、选中态和关键图表；分类使用彩色圆形图标；页面保持安静，不堆说明文字。

## 2. 页面组织

底部固定 5 个入口（二级页面除外）：

| 位置 | 名称 | 作用          |
| -- | -- | ----------- |
| 1  | 首页 | 看概览和最近记录    |
| 2  | 账单 | 查账、搜索、筛选    |
| 3  | 新增 | 记收入、支出、转账   |
| 4  | 图表 | 看统计         |
| 5  | 我的 | 账户、分类、备份、设置 |

中间“新增”是蓝色圆形按钮，只做新增记录入口，不承载其他功能。

```text
进入页：首次进入说明离线、本地、无需登录；非首次进入作为加载页
首页：概览、最近记录
账单：月份账单、搜索、详情、编辑
新增：新增收入、新增支出、新增转账
图表：收支统计、分类统计、资产统计
我的：分类管理、账户管理、数据备份、设置、关于
```

## 3. 关键流程

### 3.1 快速记一笔

```text
点新增 -> 选收入/支出/转账 -> 输入金额 -> 选分类或账户 -> 保存 -> 回到来源页并刷新
```

- 默认进入支出。
- 金额输入区永远最醒目。
- 分类使用圆形图标，常用分类优先。
- 备注默认是单行输入，不抢主要位置。
- 保存成功后回到打开新增页之前的页面。

### 3.2 查找旧账

```text
点账单 -> 按月份浏览 -> 搜索或筛选 -> 点记录 -> 查看详情 -> 编辑或删除
```

- 账单按日期分组。
- 搜索只搜备注、分类名和账户名。
- 删除必须二次确认。
- 编辑完成后回到账目详情，数据同步刷新。

### 3.3 管理账户和分类

```text
点我的 -> 账户管理或分类管理 -> 新增/编辑/停用 -> 返回列表
```

- 停用项不再出现在新建记录选择中。
- 历史记录仍显示停用项，不改名、不丢失。
- 新增表单只保留必要字段。

### 3.4 导入导出

```text
点我的 -> 数据备份 -> 导出备份或合并导入 -> 完成后给出结果
```

- 导入前必须提示“会追加到当前数据，不会自动去重”。
- 导入失败要说明原因，并保持原数据不变。

## 4. 页面规范

### 4.1 通用规则

- 使用 `KeacsScaffold`，页面背景统一为 `Background`。
- 内容左右边距 `16dp`，顶部避开状态栏，底部避开导航栏和键盘。
- 标题居中，返回按钮在左，主操作在右。
- 主内容优先用列表和分组，不把所有东西都包成卡片。
- 禁止嵌套卡片。

### 4.2 进入页

进入页用于首次进入和启动加载，视觉参考给定线稿图，但颜色必须使用当前规范。

交互规则：

- 首次进入显示标题“轻记账”、短说明、线稿账本插画、三条能力说明、“开始记账”按钮和本地保存提示。
- 点击“开始记账”后进入首页，并在本机记录已进入状态。
- 非首次进入不显示“开始记账”按钮，按钮位置显示“正在进入”和小进度提示，加载完成后自动进入首页。
- 页面不显示登录、注册、云同步、营销文案或多余说明。

### 4.3 首页

页面顺序：标题、本月结余卡片、最近记录。

交互规则：

- 结余卡片显示本月结余、收入、支出，不显示净资产。
- 统计卡片不允许点击跳转，使用轻量图标标识收入/支出，趋势按分段展示。
- 最近记录最多显示8条，点击进入详情。
- 点击“查看更多”进入账单页。
- 空状态只显示图标、标题和短提示，不显示“记一笔”按钮。

### 4.4 账单页

页面顺序：标题、搜索和更多、月份选择、当月收入支出小计、日期分组列表。

交互规则：

- 月份点击后弹出月份选择器。
- 搜索进入独立搜索页。
- 日期组头显示日期和x星期几。
- 列表项左侧是分类图标，中间是分类和备注，右侧是金额和账户。
- 向左滑动可露出删除，删除前仍需确认。

### 4.5 新增记录页

页面顺序：返回标题保存、类型分段、分类或账户、大金额输入、账户日期备注、底部数字键盘。

交互规则：

- 类型分段为支出、收入、转账，支持左右滑动切换。
- 默认进入支出。
- 支出和收入显示分类图标；转账显示转出账户和转入账户。
- 金额为空、为 0 或格式错误时，保存不可点，并在金额附近提示“金额大于0才可保存”。
- 支出和收入没有分类时，保存不可点。
- 转账账户相同时，保存不可点并提示。
- 保存中禁用重复点击，保存失败保留已输入内容。

### 4.6 分类管理页

页面顺序：返回标题编辑、支出、收入、账户分段、分类列表、底部新增分类按钮。

交互规则：

- 普通状态下点击分类进入编辑页。
- 编辑状态下显示排序和停用入口。
- 停用分类降低透明度，并标注停用。
- 被历史记录使用的分类不能直接删除，只能停用。

### 4.7 账户管理页

页面顺序：返回标题编辑、净资产卡片、账户列表、新增账户按钮。

交互规则：

- 净资产卡片显示资产、负债、净资产。
- 账户按资产、负债分组。
- 点击账户进入详情或编辑。
- 资产账户只保留直接编辑账户的功能，不提供单独的增加、减少入口。
- 停用账户降低透明度，不参与新增记录选择。

### 4.8 图表页

页面顺序：标题、支出收入资产分段、时间选择、趋势卡片、分类占比或账户排行。

交互规则：

- 默认显示支出统计。
- 图表只展示关键趋势，不做复杂金融分析。
- 排行最多显示前 5 项，其余合并为其他。
- 颜色不能作为唯一信息，必须配合文字和数值。

### 4.9 我的页和设置页

- 我的页只放分类管理、账户管理、导出备份、导入备份、设置、关于。
- 不显示登录、头像、会员、云同步。
- 设置页只放记账默认账户、默认分类、记账时间默认值、金额小数位、清除缓存。
- 清除缓存需要确认。
- 不加入首版 PRD 外的安全锁、夜间模式、联网设置。

关于页：

- 显示应用图标、应用名称、版本号。
- 提供“获取更新”入口，点击后跳转到外部发布页查看更新。
- 不在本机检查版本，不申请联网权限。

## 5. 视觉规范

颜色策略为克制型：大面积使用浅色中性背景，蓝色只用于主操作和选中态。设计源使用 OKLCH，Android 可落地为近似色值。

| 角色             | 色值        | 用途            |
| -------------- | --------- | ------------- |
| Primary        | `#3F82F6` | 主按钮、选中态、重点图表线 |
| Primary Light  | `#EAF2FF` | 选中背景、浅蓝提示     |
| Background     | `#F6F8FC` | 页面背景          |
| Surface        | `#FCFDFF` | 卡片、输入框、列表     |
| Surface Subtle | `#F1F4F8` | 搜索框、分段控件、键盘块  |
| Text Primary   | `#1F2937` | 主文字、金额        |
| Text Secondary | `#6B7280` | 副标题、说明值       |
| Text Tertiary  | `#A0A7B3` | 占位、弱信息        |
| Border         | `#E6EBF2` | 分割线、输入框边      |
| Income         | `#35C785` | 收入、正向金额       |
| Expense        | `#FF5A5F` | 支出、负向金额       |
| Warning        | `#FFB020` | 导入确认、风险提示     |
| Error          | `#E5484D` | 删除、失败         |

分类色只用于图标底色和图表，不用于大面积背景。

| 场景   | 字号     | 字重    | 行高     |
| ---- | ------ | ----- | ------ |
| 页面标题 | `18sp` | `600` | `26sp` |
| 模块标题 | `15sp` | `600` | `22sp` |
| 正文   | `14sp` | `400` | `20sp` |
| 辅助文字 | `12sp` | `400` | `17sp` |
| 大金额  | `30sp` | `700` | `38sp` |
| 卡片金额 | `24sp` | `700` | `32sp` |
| 底部导航 | `11sp` | `500` | `14sp` |

使用 Android 系统默认字体。金额使用等宽数字。禁止负字距。

| 项目      | 规范            |
| ------- | ------------- |
| 基础单位    | `4dp`         |
| 页面左右边距  | `16dp`        |
| 模块间距    | `16dp`        |
| 卡片内边距   | `16dp`        |
| 列表项最小高度 | `64dp`        |
| 卡片圆角    | `12dp`        |
| 输入框圆角   | `8dp`         |
| 主按钮圆角   | `10dp`        |
| 分类图标容器  | `40dp` 圆形     |
| 底部新增按钮  | `56dp` 圆形     |
| 最小点击区域  | `48dp x 48dp` |
| 阴影      | 只用轻阴影，不用重阴影   |

## 6. 组件和图标

| 组件               | 规范                 |
| ---------------- | ------------------ |
| `KeacsScaffold`  | 页面外壳、背景、状态栏、底部安全区  |
| `KeacsTopBar`    | 返回、标题、右侧操作         |
| `KeacsBottomBar` | 首页、账单、新增、图表、我的     |
| `OverviewCard`   | 蓝底概览卡，显示结余、收入、支出   |
| `RecordListItem` | 分类图标、标题、备注、账户、金额   |
| `CategoryIcon`   | 圆形底色、白色线性图标        |
| `SegmentedTabs`  | 白底容器、蓝色选中态         |
| `AmountInput`  | 大金额、等宽数字        |
| `NumberPad`      | 3 列数字键、删除键、确认态     |
| `FormFieldRow`   | 图标、标题、当前值、箭头       |
| `EmptyState`     | 小图标、标题、短文案，不默认放主操作 |
| `ConfirmDialog`  | 标题、正文、取消、确认        |
| `KeacsSnackbar`  | 保存失败、删除完成、导入失败     |

所有可点击组件必须有默认、按下、焦点、禁用、加载、错误状态。同类操作只使用一种样式。

图标统一使用 Compose Material Icons Rounded。常规尺寸 `24dp`，分类图标 `20dp`，放入 `40dp` 圆形底。禁止 Emoji、混用多套图标、无含义装饰图标。

业务映射：导航使用 `home`、`receipt_long`、`add`、`monitoring`、`person`；操作使用 `search`、`more_horiz`、`AutoMirrored.Rounded.ArrowBack`；分类使用餐饮 `restaurant`、交通 `directions_bus`、购物 `shopping_bag`、住房 `home`、娱乐 `sports_esports`、医疗 `local_hospital`、教育 `school`、通讯 `phone_android`、日用 `local_mall`、工资 `work`、其他 `category`。

## 7. 状态和表单

| 状态    | 规范                 |
| ----- | ------------------ |
| 加载    | 使用骨架或小进度，不遮挡已有内容   |
| 空首页   | 显示“暂无记录”和短提示，不显示按钮 |
| 空账单   | 显示“暂无记录”和短提示，不显示按钮 |
| 空分类   | 显示“新增分类”           |
| 空账户   | 显示“新增账户”           |
| 保存失败  | Snackbar，保留表单      |
| 删除成功  | Snackbar           |
| 导入失败  | 弹窗说明原因             |
| 文件无权限 | 弹窗说明无法读取或保存        |
| 大字体   | 金额不截断，按钮不挤压        |

表单规则：

- 金额必须大于 0，最多保留两位小数。
- 日期默认当前时间。
- 支出和收入必须选择分类。
- 转账必须选择两个不同账户。
- 保存中禁止重复提交。
- 保存失败不能清空用户输入。
- 删除、导入、清除缓存必须确认。
- 输入错误要就近显示，不只在页面顶部提示。

## 8. 动效和无障碍

| 场景   | 时长      | 动效                     |
| ---- | ------- | ---------------------- |
| 页面进入 | `220ms` | 淡入加轻微位移                |
| 页面退出 | `180ms` | 淡出加轻微位移                |
| 列表增删 | `180ms` | Compose LazyList 项位移动画 |
| 分段切换 | `160ms` | 背景滑动、文字变色              |
| 按钮按下 | `80ms`  | 涟漪或透明度反馈               |
| 图表加载 | `300ms` | 线条或透明度渐入               |
| 弹窗   | `180ms` | 淡入、轻微缩放                |

动效只用于反馈状态，不做循环装饰动画。遵循系统动画缩放设置。

无障碍规则：点击区域不小于 `48dp x 48dp`；文字对比度不低于 `4.5:1`；图标按钮必须有 `contentDescription`；金额颜色必须配合正负号或文字；TalkBack 顺序与视觉顺序一致；支持系统大字体；键盘弹出时，正在编辑的输入框不能被遮挡。

## 9. Compose 落地约束

- 使用 Jetpack Compose + Material 3。
- 主题入口统一为 `KeacsTheme`。
- 颜色、字体、圆角、间距从主题或设计 token 获取。
- 业务页面不得硬编码颜色和尺寸。
- 页面状态使用不可变 `UiState`。
- 列表使用稳定 `key`。
- 导航使用类型安全路由。
- ViewModel 发出一次性导航事件。
- 预览至少覆盖默认、大字体、空状态、错误状态。

## 10. 验收清单

- 底部导航、顶部栏、卡片、列表项风格一致。
- 首页能快速进入记账、账户、图表和更多。
- 新增页优先输入金额，保存规则清楚。
- 账单页能按月份、日期、搜索找到记录。
- 分类、账户、统计、设置都有明确入口。
- 没有登录、联网、云同步入口。
- 没有文字重叠、按钮溢出、横向滚动。
- 保存失败、删除确认、导入失败有明确反馈。
- 图表简单可读，不做复杂金融分析。
- 支持 `360dp` 宽度、小屏和大字体。
