# 助手对话架构评估

日期：2026-05-13

## 评估结论

助手不可用的主要原因不是单个页面问题，而是“模型结果、后端兜底、App 本地兜底、确认卡片”四段链路没有形成稳定闭环。

## 发现的问题

1. 后端默认模型模式会返回“已收到”一类回声，查账、改单、定时记账等真实任务没有可用结果。
2. 后端流式接口没有复用统一的助手处理逻辑，容易和普通聊天接口表现不一致。
3. 模型失败或网络失败时，App 本地兜底只覆盖新增记账和简单统计，删账、改单、定时记账缺口明显。
4. 定时记账语句里，“每月1号”的“1”会被误判为金额。
5. 确认卡片内部存在纵向滚动，信息密度偏高；字段修改入口也不够直接。

## 修复方向

1. 后端统一走助手编排逻辑，模型不可用时使用本地规则给出可读结果。
2. 强化后端和 App 两侧兜底，覆盖新增、查询、修改、删除、定时记账和风险边界。
3. 用 150 条真实对话场景做回放测试，覆盖短对话、长对话、复杂任务和异常场景。
4. 确认卡片改成无内部滚动的字段展示，点击字段行直接打开底部弹窗修改。
