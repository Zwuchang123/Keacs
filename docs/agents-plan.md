# Keacs Agent 助手开发阶段表

## 使用规则

本文件只跟踪 Keacs Agent 助手的开发阶段和阶段回填。

每完成一个阶段后，必须回填该阶段的`状态`、`完成时间`和`回填记录`。状态只使用：`未开始`、`进行中`、`已完成`、`阻塞`。

## 开发阶段

| 阶段 | 状态 | 目标 | 开发范围 | 验收要点 | 完成时间 | 回填记录 |
| --- | --- | --- | --- | --- | --- | --- |
| 1. 官方服务端基础和模型通路 | 未开始 | 建立官方在线助手服务基础，让 App 后续可以稳定访问官方模型服务。 | `server/`、FastAPI 项目结构、健康检查、设备识别、限流、日志、模型服务客户端、结构化返回校验、后端测试。 | `GET /health` 正常；官方服务可用 mock 模型返回结构化结果；限流和错误请求有明确返回；日志不包含用户完整输入、账本明细、API Key 等敏感信息。 | - | - |
| 2. App 在线助手配置和网络接入 | 未开始 | 让用户可以启用在线助手，并选择官方服务或自定义模型服务。 | 设置页、DataStore 配置、Android 网络权限、Agent 配置模型、Agent 网络客户端、配置校验。 | 未启用时不发起 Agent 网络请求；官方模式和自定义模式配置规则清晰；自定义模式缺少访问地址或 API Key 时有提示；连接失败不影响本地记账；API Key 不写入日志。 | - | - |
| 3. 助手入口和页面骨架 | 未开始 | 将“助手”作为底部一级页面，并把原“发现”内容迁入“我的”。 | 底部导航、`ui/agent` 页面、助手 ViewModel 基础状态、消息列表、输入框、发送状态、错误状态、“我的”页承接原发现内容。 | 底部导航为“首页、统计、新增、助手、我的”；点击“助手”进入助手页；“我的”仍能查看原发现页核心信息；小屏和大字体下不遮挡、不溢出。 | - | - |
| 4. 本地上下文和查账问答 | 未开始 | 助手能基于最小必要本地数据回答账本问题，查询类问题不产生写入。 | `domain/agent`、`data/agent`、AgentContextProvider、分类/账户/候选账目/月度统计/余额摘要读取、对话结果渲染。 | 可回答月度支出、分类支出、账户余额等问题；回答中说明数据范围；查账过程不会新增、修改或删除本地数据；只上传本次问题需要的数据。 | - | - |
| 5. 写入类操作确认执行 | 未开始 | 让新增、修改、删除、批量处理和定时记账都通过预览与确认后执行。 | Agent 操作模型、候选选择卡片、操作预览卡片、确认弹窗、现有 UseCase 调用、执行结果反馈。 | 写入前必须展示预览；用户确认后才写入；用户取消后数据不变；新增、修改、删除、批量修改、定时记账主路径可用；余额和统计在写入后正确刷新。 | - | - |
| 6. 轻量建议和安全边界 | 未开始 | 支持消费复盘、预算提醒和省钱建议，同时避免高风险财务建议。 | 后端提示词、输出校验、App 错误状态、边界提示、异常处理。 | 可回答消费复盘和轻量建议；遇到投资、借贷、保险等问题不直接给决策结论；数据不足时明确说明限制；模型异常、超时和返回格式错误都有可读提示。 | - | - |
| 7. 联调验收和收口 | 未开始 | 打通官方服务、自定义模型服务和 App 主路径，确认首版开发任务完整可用。 | 服务端部署配置、HTTPS、环境变量、真机联调、Android 与后端回归测试、阶段问题修复。 | 官方服务可通过 HTTPS 健康检查；App 可连接官方服务和自定义服务；错误配置会被拒绝或提示；API Key 不出现在 App 日志、后端日志和仓库中；本阶段必测项通过后回填结果。 | - | - |

## 阶段执行要求

| 要求 | 说明 |
| --- | --- |
| 阶段顺序 | 原则上按 1 到 7 执行；只有不依赖前序结果的 UI 骨架或测试准备可以并行。 |
| 回填方式 | 阶段完成后，把`状态`改为`已完成`，填写完成时间，并在`回填记录`中写本阶段完成内容、验证结果和遗留阻塞。 |
| 阻塞处理 | 阶段被阻塞时，把`状态`改为`阻塞`，在`回填记录`中写清楚阻塞原因和需要谁决策。 |
| 测试记录 | 只记录通过或失败数量，不逐条列出用例。 |
| 文档同步 | 阶段开发中如果需求、架构或路径发生变化，只更新对应主文档；本文件只回填阶段执行结果。 |
