# 待办模块 PRD

## 1. 模块定位

待办模块是 Keacs 平台的低风险验证模块，用于验证非财务类功能如何接入 Agent、公共组件和模块契约。

模块目标：

- 快速捕捉、整理和完成个人待办。
- 让 Agent 能从对话中创建、查询和总结待办。
- 复用统一输入、消息卡片、确认卡片和提醒组件。

## 2. 用户场景

- 随手记录今天要做的事。
- 把自然语言拆成待办标题、日期、优先级和备注。
- 查看今日待办、逾期待办和已完成事项。
- 晚间复盘时同步查看账本和待办完成情况。

## 3. 功能范围

- 新增、编辑、完成、取消完成和删除待办。
- 支持日期、优先级、备注和完成状态。
- 支持今日、最近 7 天、逾期和全部待办查询。
- Agent 可生成待办草稿，写入前进入确认卡片。

## 4. Agent 工具边界

- 查询类工具返回待办摘要和候选列表。
- 写入类工具返回待办操作草稿，不直接写入。
- 批量删除或批量完成必须展示影响条数。

## 5. 复用公共组件

- AgentInputComposer：用于自然语言捕捉。
- AgentMessageCard：用于待办总结和错误提示。
- ActionPreviewCard：用于创建、修改、删除和批量完成确认。
- TaskStageCard：用于长任务整理阶段。
- 列表组件、空态组件、摘要卡片和权限门禁。

## 6. 非目标

- 不做团队协作、多人分派或项目管理。
- 不做复杂周期任务和甘特图。
- 不接入外部日历写入，除非后续单独立项。

## 7. 验收标准

- 用户能完成待办新增、编辑、完成和删除主路径。
- Agent 生成的待办写入必须先确认。
- 待办模块不直接访问记账模块数据库。
- 公共组件在待办模块中不复制实现。
