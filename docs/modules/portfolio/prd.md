# 资产观察模块 PRD

## 1. 模块定位

资产观察模块用于记录和观察用户关心的资产信息，帮助用户理解个人资产结构和变化。模块只做记录、统计、提醒和信息摘要，不做交易、不做个性化投资建议。

模块目标：

- 记录用户手动维护的资产观察项。
- 展示资产分布、变动和提醒。
- 让 Agent 能整理资产观察摘要和风险提示。

## 2. 用户场景

- 记录基金、股票、现金、债务或其他资产观察项。
- 查看资产观察摘要和变化备注。
- 让 Agent 解释某个观察项的公开信息或历史记录。
- 根据用户输入创建提醒草稿，例如“关注年报”“下周复查”。

## 3. 功能范围

- 手动新增、编辑、删除观察项。
- 记录名称、类型、备注、观察金额、更新时间和提醒。
- 展示资产观察摘要、分布和最近变化。
- Agent 可生成信息摘要、风险提示和提醒草稿。

## 4. Agent 工具边界

- 可以整理用户手动记录和公开信息摘要。
- 可以提示风险、波动和数据来源限制。
- 不能推荐买入、卖出、加仓、减仓或承诺收益。
- 不能执行交易、连接交易账户或托管资产。

## 5. 复用公共组件

- Markdown 渲染：用于公开信息摘要和风险说明。
- AgentMessageCard：用于资产观察问答。
- ActionPreviewCard：用于创建提醒或保存观察备注。
- 趋势组件、金额组件、摘要卡片、错误提示和权限门禁。

## 6. 非目标

- 不做证券交易、加密货币交易或自动调仓。
- 不做个性化投资建议、收益预测、税务或法律建议。
- 不接入真实券商账户或银行账户。

## 7. 验收标准

- 所有资产数据由用户手动维护或明确来源。
- Agent 回答必须避免投资建议，并提示信息仅供记录和观察。
- 保存提醒或备注必须先确认。
- 模块文案使用“资产观察”，不使用“投资顾问”或“交易”定位。
