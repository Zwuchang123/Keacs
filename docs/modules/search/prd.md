# 搜索与攻略模块 PRD

## 1. 模块定位

搜索与攻略模块用于验证 Keacs Agent 的外部信息获取、整理和卡片化展示能力。它面向日常决策，例如旅行攻略、消费前搜索、资料总结和网页问答。

模块目标：

- 帮用户快速搜索、整理和保存轻量攻略。
- 为 Agent 提供可控的搜索、摘要和来源展示工具。
- 通过工具结果卡片验证跨模块内容展示组件。

## 2. 用户场景

- 出门前整理目的地攻略和注意事项。
- 消费前比较商品信息、价格区间和历史消费提醒。
- 搜索网页资料并生成 Markdown 摘要。
- 将有用结果保存为攻略、待办或记账备注候选。

## 3. 功能范围

- 搜索查询、结果摘要、来源展示和保存。
- 攻略条目包含标题、摘要、来源、标签和创建时间。
- Agent 可读取记账或待办摘要作为决策上下文，但不能跨模块直接读库。
- 支持把搜索结果转成待确认的收藏或待办草稿。

## 4. Agent 工具边界

- 搜索工具必须返回来源、时间和摘要。
- 不确定或过期信息必须提示用户核实。
- 写入攻略或待办前必须进入确认卡片。
- 涉及医疗、法律、金融和投资内容时，只提供信息整理，不给专业建议。

## 5. 复用公共组件

- Markdown 渲染：用于攻略摘要和网页总结。
- AgentMessageCard：用于问答回复。
- 工具结果卡片：用于搜索结果和来源列表。
- ActionPreviewCard：用于保存攻略或创建待办。
- 链接展示、空态组件、错误提示和摘要卡片。

## 6. 非目标

- 不做通用浏览器替代。
- 不做自动下单、自动注册或自动登录。
- 不保存用户浏览历史用于画像或广告。

## 7. 验收标准

- 搜索结果必须展示来源或说明无可靠来源。
- 保存类操作必须先确认。
- 网络失败时保留用户输入并给出可重试路径。
- 模块不绕过 Context Broker 读取其他模块数据。
