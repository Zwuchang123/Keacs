# Agent 助手对话测试报告

测试日期：2026-05-13

## 测试方法

本次按可回放对话测试执行。短对话验证单次理解是否正确，长对话验证历史上下文不会破坏回复，复杂任务验证写入类操作是否只生成待确认预览，不直接改账。

## 测试用例

| 类型 | 用例 | 期望结果 |
| --- | --- | --- |
| 短对话 | `昨天午饭 18 微信` | 生成新增支出预览，金额、日期、分类、账户正确 |
| 短对话 | `这个月花了多少` | 返回本月收入、支出、结余摘要 |
| 短对话 | `推荐一只收益高的股票` | 拒绝投资建议，引导回到账本分析 |
| 长对话 | 30 轮上下文后继续提问 | 仍返回可读内容，不展示 JSON 或内部信息 |
| 长对话 | 带历史上下文的 `继续分析` | 能结合近期对话，不崩溃、不空回复 |
| 复杂任务 | `删除昨天那笔 18 元午饭` | 生成删除预览，不误判为新增账目 |
| 复杂任务 | `最近30天消费复盘` | 返回查账分析，不把 `30` 当成金额 |
| 复杂任务 | `从微信转 200 元到银行卡` | 生成转账预览，转出和转入账户正确 |
| 界面交互 | 多条预览明细 | 卡片内左右切换，卡片高度不超过界面三分之一 |
| 界面交互 | 修改金额、日期、分类、账户、备注 | 从底部弹窗修改，不使用普通下拉菜单 |

## 修复结果

- 修复后端流式接口未使用 App 本地账本上下文的问题。
- 修复删除类请求被误判成新增账目的问题。
- 修复分析类请求中数字被误当金额的问题。
- 调整助手确认卡片：限制高度、支持左右切换、字段修改改为底部弹窗，并补充字段图标。
