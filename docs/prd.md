# 轻量记账 App PRD

<br />



## 1. 产品定位

- 产品名称：Keacs。
- 面向对象：个人自用记账用户。
- 核心目标：简单、轻量、离线可用，打开就能记。
- 产品原则：优先满足快速记账、看懂收支、稳定保存，不引入复杂财务模型。

当前实现以 Android 版本为准，不包含账号、云同步、多人协作和复杂预算体系。

## 2. 当前版本范围

当前版本已经落地的核心范围如下：

- 首次欢迎引导。
- 首页概览、按月查看和每日账单明细。
- 发现页当月概览与资产负债概览。
- 新增和编辑支出、收入、转账。
- 收入分类、支出分类、账户分类管理。
- 资产账户、负债账户管理。
- 收入、支出、结余统计图表。
- 本地 JSON 备份导出与合并导入。
- 基础设置、关于页、更新链接。

## 4. 使用流程

### 4.1 首次使用

用户首次打开 App 时，先看到欢迎页。欢迎页展示：

- 产品名称“轻记账”
- 副标题“轻量 · 简单 · 离线记账”
- 插画
- “开始记账”按钮
- “数据仅保存在本设备，不会上传或同步任何云端”提示

点击“开始记账”后写入本机标记，随后进入首页。

### 4.2 再次打开

同一设备后续再次打开 App 时，不再展示欢迎页。应用完成本地初始化后，直接进入首页。

### 4.3 日常使用

1. 在首页查看选中月份的结余、收入、支出和每日账单明细。
2. 在底部点击“新增”进入记账页。
3. 选择“支出”“收入”或“转账”。
4. 选择分类或账户，输入金额、日期、备注后保存。
5. 在统计页查看趋势和排行，在发现页查看当月概览和资产负债。
6. 在“我的”页管理分类、账户或进行备份迁移。

## 5. 首页

首页用于按月查看概览和每日账单明细。

### 5.1 展示内容

- 选中月份
- 选中月份结余
- 选中月份收入
- 选中月份支出
- 按日期分组的每日账单明细

### 5.2 月份筛选

- 支持查看最近 10 年内的月份，且不能选择未来月份。
- 切换月份后，首页概览和每日账单明细同步刷新。

### 5.3 每日账单明细

- 列表按日期分组，组内按时间倒序展示。
- 每日分组只显示月日和周几。
- 每日分组展示当天支出合计。
- 支持点击单条记录进入账目详情。
- 没有记录时展示空态。

### 5.4 当前范围说明

首页不直接展示总资产、总负债和净资产。

## 6. 发现页

发现页用于承载轻量概览和后续低频能力预留。

### 6.1 展示内容

- 当月收入
- 当月支出
- 当月结余
- 月份选择器
- 总资产
- 总负债
- 净资产
- “更多功能敬请期待~”提示

### 6.2 统计规则

- 当月收入、支出和结余按选中月份统计，月份不能选择未来月份。
- 总资产、总负债和净资产按当前启用账户余额统计。

## 7. 记账需求

### 7.1 记录类型

当前版本支持三类记录：

- 支出
- 收入
- 转账

### 7.2 通用字段

三类记录的共同输入包括：

- 金额
- 日期
- 备注

规则：

- 金额必须大于 0 才能保存。
- 日期只能选择当前或过去时间，不能选择未来日期。
- 备注为选填。
- 编辑记录时沿用原始数据进行回填。

### 7.3 支出记录

字段：

- 金额
- 支出分类
- 支出账户
- 日期
- 备注

规则：

- 分类必填。
- 账户可不选；如已在设置中配置默认记账账户，进入时会优先带出。
- 保存后对应账户余额减少。
- 删除或编辑后，余额和统计自动修正。

### 7.4 收入记录

字段：

- 金额
- 收入分类
- 收入账户
- 日期
- 备注

规则：

- 分类必填。
- 账户可不选；如已在设置中配置默认记账账户，进入时会优先带出。
- 保存后对应账户余额增加。
- 删除或编辑后，余额和统计自动修正。

### 7.5 转账记录

字段：

- 金额
- 转出账户
- 转入账户
- 日期
- 备注

规则：

- 转出账户必填。
- 转入账户必填。
- 转出账户和转入账户不能相同。
- 转账不计入收入或支出统计，只影响账户余额。
- 删除或编辑后，余额自动修正。

### 7.6 金额输入

当前金额键盘支持：

- 数字输入
- 小数点
- 退格
- `+` 和 `-` 连续计算

规则：

- 最多保留 2 位小数。
- 输入总长度上限为 11 个字符。
- 表达式结果必须大于 0 才可保存。
- 例如 `10+2.50` 可以直接计算为 `12.50`。

### 7.7 账目详情

账目详情页当前展示：

- 记录类型/分类信息
- 金额
- 账户信息
- 账目日期
- 备注

并提供：

- 编辑记录
- 删除记录

当前详情页不展示创建时间和更新时间。

### 8.4 预置账户分类

- 微信

- 银行卡
- 公积金
- 投资账户

负债账户分类：

- 消费贷
- 房贷车贷

### 8.5 分类规则

- 分类名称不能为空。
- 分类名称最多 4 个字。
- 同一方向下不允许重名。
- 分类支持新增、编辑、停用。
- 没有历史记录的分类可以删除。
- 已被历史记录使用的分类不能删除，只能停用。
- 停用后不再用于新建记录，但历史记录继续正常显示。

### 9.1 账户范围

账户当前只区分两种性质：

- 资产账户
- 负债账户

### 9.2 预置账户

资产账户：

- 支付宝
- 微信
- 现金
- 银行卡
- 公积金
- 投资账户
- 其他资产

负债账户：

- 信用卡
- 花呗/白条
- 消费贷
- 房贷/车贷
- 亲友借款
- 其他负债

### 9.3 列表页内容

账户管理页当前展示：

- 净资产
- 总资产
- 总负债
- 资产账户列表
- 负债账户列表

### 9.4 账户字段

- 账户名称
- 账户性质
- 账户类型
- 当前余额
- 是否启用

### 9.5 账户规则

- 账户名称不能为空。
- 全局不允许账户重名。
- 负债账户余额统一按负数存储和展示。
- 账户余额通过编辑页直接修改，不单独提供“调增/调减”功能。
- 没有历史记录的账户可以删除。
- 已被历史记录使用的账户不能删除，只能停用。
- 停用账户不会从历史记录中消失。

### 9.6 特殊账户说明

当前版本只处理余额，不处理本金、持仓、账单日、免息期、手续费、冻结金额等复杂信息。

## 10. 统计页

### 10.1 当前结构

统计页位于底部导航第二个入口，当前分为 3 个标签：

1. 支出
2. 收入
3. 结余

### 10.2 时间维度

当前支持：

- 按月查看
- 按年查看

时间选择器只能选择当前或过去的年月。

### 10.3 展示内容

统计页顶部统一展示：

- 收入
- 支出
- 结余

不同标签页内容如下：

- 支出：支出趋势、支出排行榜
- 收入：收入趋势、收入排行榜
- 结余：结余趋势

支出、收入、结余分段位于统计卡片下方、趋势图上方。

### 10.4 统计规则

- 收入统计只计算收入记录。
- 支出统计只计算支出记录。
- 结余按收入减支出计算。
- 转账不计入收入、支出和结余统计。
- 已停用分类对应的历史记录仍参与统计。
- 趋势图会补齐已发生时间节点，没有记录的节点记为 0。
- 当前统计页不展示账户余额排行、总资产、总负债和净资产图表。

## 12. 数据保存与迁移

### 12.1 保存原则

- 所有数据只保存在本地设备。
- 不依赖网络和账号系统。
- 当前只支持人民币记账。

### 12.2 导出

当前支持导出单个 JSON 备份文件，备份内容包括：

- 分类
- 账户
- 记录
- 备份版本
- 导出时间
- App 版本号
- 余额符号规则

### 12.3 导入

当前只支持合并导入。

导入规则：

- 记录直接追加，不做去重。
- 分类按“方向 + 名称”复用已有项，不重复插入。
- 账户始终新增；如名称冲突，会自动重命名为“原名称（导入2）”等形式。
- 支持读取备份版本 1 和版本 2。
- 导入旧备份时，会把负债账户余额转换为当前负数规则。

### 12.4 换机迁移

用户可在旧设备导出，在新设备导入，恢复分类、账户和记录数据。

## 13. 核心业务规则

### 13.1 余额规则

- 资产账户以正数表示。
- 负债账户以负数表示。
- 收入记入账户时加余额。
- 支出记入账户时减余额。
- 转账按“转出减、转入加”处理。
- 账户性质不会改变收支方向，只影响余额符号展示。

### 13.2 历史一致性

- 新增、编辑、删除记录后，账户余额必须同步修正。
- 停用的账户和分类，其历史记录仍要可读。
- 统计结果始终以当前有效记录为准。
