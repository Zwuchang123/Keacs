## 1. 产品定位

- 产品名称：Keacs。
- 面向对象：个人自用记账用户。
- 核心目标：简单、轻量、离线可用，打开就能记。
- 产品原则：优先满足快速记账、看懂收支、稳定保存，不引入复杂财务模型。

当前实现以 Android 版本为准。基础记账能力不依赖账号、云同步、多人协作和复杂预算体系。Agent 助手是可选在线能力，首版不包含账号登录和云同步，用户不开启在线助手时，原有离线功能必须完整可用。

## 2. 当前版本范围

当前离线基础版本已经落地的核心范围如下：

- 首次欢迎引导。
- 首页概览、按月查看和每日账单明细。
- 我的页当月概览与资产负债概览。
- 新增和编辑支出、收入、转账。
- 收入分类、支出分类、账户分类管理。
- 资产账户、负债账户管理。
- Excel 账目添加。
- 定时记账。
- 收入、支出、结余统计图表。

Agent 助手作为独立开发任务，新增范围如下：

- 底部一级“助手”页面。
- 原“发现”页内容迁入“我的”。

- 消费复盘、预算提醒、省钱建议等轻量建议。

## 4. 使用流程

### 4.1 首次使用

用户首次打开 App 时，先看到欢迎页。欢迎页展示：

- 产品名称“轻记账”
- 副标题“轻量 · 简单 · 离线记账”
- 插画
- “开始记账”按钮
- “数据仅保存在本设备，不会上传或同步任何云端”提示

点击“开始记账”后写入本机标记，随后进入首页。

### 4.2 再次打开

同一设备后续再次打开 App 时，不再展示欢迎页。应用完成本地初始化后，直接进入首页。

### 4.3 日常使用

1. 在首页查看选中月份的结余、收入、支出和每日账单明细。
2. 在底部点击“新增”进入记账页。
3. 选择“支出”“收入”或“转账”。
4. 选择分类或账户，输入金额、日期、备注后保存。
5. 在统计页查看趋势和排行，在“助手”页通过对话处理记账和查账，在“我的”页查看当月概览和资产负债。
6. 在“我的”页管理分类、账户或进行备份迁移。
7. 可从 Excel 添加已有账目，也可设置周、月、年的定时记账。

### 4.4 在线助手使用

1. 用户先在设置页启用在线助手，并选择模型服务模式。
2. 用户从底部导航进入“助手”页。
3. 用户输入自然语言，例如“昨天午饭 18 微信”或“这个月餐饮花了多少”。
4. 助手根据问题请求最小必要的本地上下文。
5. 查询类问题直接展示回答。
6. 写入类问题先展示操作预览。
7. 用户确认后，应用按现有记账规则保存、修改或删除数据。
8. 用户取消时，不写入任何数据。
9. 在线助手不可用时，页面展示原因和下一步，不影响本地记账。

## 5. 首页

首页用于按月查看概览和每日账单明细。

### 5.1 展示内容

- 选中月份
- 选中月份结余
- 选中月份收入
- 选中月份支出
- 按日期分组的每日账单明细

### 5.2 月份筛选

- 支持查看最近 10 年内的月份，且不能选择未来月份。
- 切换月份后，首页概览和每日账单明细同步刷新。

### 5.3 每日账单明细

- 列表按日期分组，组内按时间倒序展示。
- 每日分组只显示月日和周几。
- 每日分组展示当天支出合计。
- 支持点击单条记录进入账目详情。
- 没有记录时展示空态。

### 5.4 当前范围说明

首页不直接展示总资产、总负债和净资产。

## 6. 助手页

助手页是 Agent 助手的一级入口，用于通过对话完成记账、查账、改账、删账、定时记账、消费复盘和轻量建议。

助手页是独立开发任务的核心页面。开发时优先阅读本章节。

### 6.1 页面状态

助手页需要覆盖以下状态：

- 未启用：提示用户到设置页启用在线助手。
- 未配置：提示用户选择模型服务模式；自定义模式下提示填写访问地址和 API Key。
- 空会话：展示简短欢迎语和常用示例。
- 正常对话：展示用户消息、助手回复和操作卡片。
- 发送中：展示轻量加载状态，禁止重复发送同一条消息。
- 等待确认：展示待执行操作，等待用户确认或取消。
- 执行成功：展示结果，并刷新相关本地数据。
- 执行失败：说明失败原因，保留用户输入和上下文。
- 服务器不可用：提示网络或服务器问题，不影响本地记账。
- 长对话提醒：对话内容过长时提示用户清空，避免旧上下文影响判断。

### 6.2 空会话引导

空会话只做轻量引导，不堆说明文字。

建议展示 3 个示例：

- “记一笔午饭 18 元”
- “这个月花了多少？”
- “帮我看看本月支出”

用户点击示例后，将示例填入输入框或直接发送，具体交互按实现成本选择一种即可。

已有对话再次进入时，需要保留原对话，同时继续在底部提供轻量引导示例，方便用户继续追问或开始新的记账请求。

### 6.3 对话输入

首版只支持文字输入。

输入规则：

- 输入不能为空。
- 发送中禁止重复点击发送。
- 发送失败后保留输入内容。
- 单条输入需要限制最大长度，避免一次发送过长内容。
- 输入框需要支持多行自动换行，内容过多时不横向挤压。
- 不支持语音、图片、票据识别、文件上传。

用户点击发送前，不上传任何账本数据。

### 6.4 消息类型

助手页至少支持以下消息类型：

- 用户消息：展示用户输入的原文。
- 助手文本回复：用于回答查账、解释结果和追问。
- 操作预览卡片：用于展示待确认的新建、修改、删除、批量处理和定时记账。
- 候选选择卡片：用于展示多条可能匹配的账目，让用户选择。
- 执行结果消息：用于展示保存成功、修改成功、删除成功或执行失败。
- 错误消息：用于展示网络、配置、模型和本地校验异常。

消息按时间从上到下展示，进入页面后默认定位到最新消息。

消息展示规则：

- 消息卡片根据内容自适应宽高，只设置最大宽度。
- 助手回复支持换行和重点加粗，避免整段文字难以阅读。
- 发送中和回复完成后展示本次响应耗时。

### 6.5 记账理解规则

助手从用户输入中识别以下信息：

- 记账类型：支出、收入、转账。
- 金额。
- 日期。
- 分类。
- 账户。
- 备注。

默认规则：

- 未说日期时，默认使用今天。
- 日期不能是未来日期。
- 金额必须大于 0，最多保留 2 位小数。
- 能明确识别分类时自动带出分类。
- 收入和支出未说账户时，优先使用设置中的默认记账账户；没有默认账户时可留空。
- 转账必须识别转出账户和转入账户；缺失时必须追问。
- 无法明确识别金额、类型或分类时，必须追问，不能直接保存。
- 用户原话中有可读备注时，可写入备注；没有备注时留空。

示例：

- “昨天午饭 18 微信”识别为支出，分类为餐饮，账户为微信，日期为昨天。
- “今天工资 8000 到银行卡”识别为收入，分类为工资，账户为银行卡。
- “从微信转 200 到银行卡”识别为转账，转出账户为微信，转入账户为银行卡。

### 6.6 查账和分析

助手支持回答账本相关问题。

支持范围：

- 月度收入、支出、结余。
- 分类支出或收入汇总。
- 最近一段时间的大额支出。
- 指定账户余额。
- 消费习惯复盘。
- 预算提醒。
- 省钱建议。

查账优先按用户问题选择数据范围，不固定只看本月。常见范围包括今天、昨天、本周、最近 7 天、最近 30 天、最近 3 个月、最近半年、最近一年、今年、去年、本月、上月和全部账单。

用户明确提出“全部账单”“所有历史”“累计”“从开始到现在”等长期分析诉求时，允许把全部账单明细上传给模型分析。用户只问单日、单周、单月或单年问题时，只上传对应范围的账单和统计摘要。用户没有明确要求明细分析时，优先上传统计摘要，避免无关数据进入模型上下文。

助手回答时需要说明数据范围，例如“按 2026 年 5 月统计”或“按最近 7 天统计”。

### 6.7 改账和候选选择

用户要求修改账目时，助手需要先定位候选账目。

候选账目展示内容：

- 日期
- 记账类型
- 金额
- 分类
- 账户
- 备注

规则：

- 只匹配到 1 条时，可直接生成修改预览。
- 匹配到多条时，必须让用户选择。
- 匹配不到时，提示用户补充日期、金额、分类或账户。
- 修改后仍必须符合记账规则。

示例：

- “把昨天午饭改成 20”
- “上周那笔打车分类改成交通”
- “把工资账户改成银行卡”

### 6.8 删账

用户要求删除账目时，助手需要先定位候选账目并展示删除预览。

规则：

- 删除必须二次确认。
- 删除预览必须展示将删除的账目数量和明细。
- 匹配到多条时，必须让用户选择或确认批量删除。
- 用户取消后，不删除任何数据。

示例：

- “删掉昨天重复的午饭”
- “删除今天那笔 18 元支出”

### 6.9 批量处理

助手支持有限批量处理。

支持范围：

- 批量修改分类。
- 批量修改账户。
- 批量追加或修改备注。

规则：

- 批量处理必须展示影响条数和明细列表。
- 批量处理必须用户确认后执行。
- 影响范围无法确定时必须追问。
- 首版不支持批量新增大量账目。

示例：

- “把上周所有打车都归到交通”
- “把本月备注里有咖啡的支出改成餐饮”

### 6.10 定时记账

助手支持创建、修改、停用定时记账。

创建和修改定时记账时，需要展示：

- 收支类型
- 重复周期
- 金额
- 分类
- 账户
- 转出账户
- 转入账户
- 下次生成时间
- 备注
- 是否启用

定时记账仍沿用 `7.8 定时记账` 的规则。

示例：

- “每月 1 号房租 2500，从银行卡支出”
- “每周一记一笔早餐 10 元”
- “停用房租定时记账”

### 6.11 操作确认

所有会改变本地数据的操作都必须用户确认后执行。

写入类操作包括：

- 新增账目
- 修改账目
- 删除账目
- 批量修改
- 创建定时记账
- 修改定时记账
- 停用定时记账

操作预览必须展示：

- 操作类型
- 影响对象
- 金额
- 日期
- 分类
- 账户
- 备注
- 影响条数
- 风险提示

用户确认后，应用按现有业务规则执行。用户取消后，不写入任何数据。

确认交互规则：

- 操作预览卡片必须直接提供“确认执行/确认删除”和“取消”按钮。
- 预览内容必须使用普通用户能看懂的账目格式，不能直接展示 JSON。
- 用户取消或确认后，需要移除对应待确认动作，避免重复执行。

### 6.12 轻量建议边界

助手可以提供：

- 消费习惯复盘。
- 预算提醒。
- 支出异常提醒。
- 省钱建议。
- 分类使用建议。

助手禁止提供：

- 投资建议。
- 借贷建议。
- 保险建议。
- 税务、法律、医疗等专业建议。
- 承诺收益或预测收益。

遇到禁止范围时，助手需要说明无法提供该类建议，并可引导用户回到账本记录和消费复盘。

### 6.13 异常处理

助手需要处理以下异常：

- 未启用在线助手。
- 自定义模式未配置访问地址。
- 自定义模式未配置 API Key。
- 服务器不可达。
- API Key 无效或模型服务拒绝访问。
- 模型调用失败。
- 模型返回内容无法解析。
- 模型响应较慢。
- 本地业务校验失败。
- 候选账目不唯一。
- 分类或账户不存在。
- 日期为未来时间。
- 金额不合法。

异常提示必须说明原因和下一步，不使用错误码作为主要文案。

模型响应较慢或返回不稳定时，不应只展示“模型服务不可用”。能用本地规则判断的常见记账和查账请求，需要先给出可读结果或待确认预览；不能可靠判断时，再提示用户补充信息或稍后重试。

### 6.14 会话保留与清理

- 助手需要保留最近对话，用户退出后再次进入仍能看到原对话。
- 请求模型时需要携带必要的近期对话，让模型能理解追问和多轮补充。
- 对话保存只保存在本机，不上传完整历史账本。
- 用户可以手动清空对话。
- 对话过长时需要提醒用户清理，避免旧内容影响新的判断。

## 7. 记账需求

### 7.1 记录类型

当前版本支持三类记录：

- 支出
- 收入
- 转账

### 7.2 通用字段

三类记录的共同输入包括：

- 金额
- 日期
- 备注

规则：

- 金额必须大于 0 才能保存。
- 日期只能选择当前或过去时间，不能选择未来日期。
- 备注为选填。
- 编辑记录时沿用原始数据进行回填。

### 7.3 支出记录

字段：

- 金额
- 支出分类
- 支出账户
- 日期
- 备注

规则：

- 分类必填。
- 账户可不选；如已在设置中配置默认记账账户，进入时会优先带出。
- 保存后对应账户余额减少。
- 删除或编辑后，余额和统计自动修正。

### 7.4 收入记录

字段：

- 金额
- 收入分类
- 收入账户
- 日期
- 备注

规则：

- 分类必填。
- 账户可不选；如已在设置中配置默认记账账户，进入时会优先带出。
- 保存后对应账户余额增加。
- 删除或编辑后，余额和统计自动修正。

### 7.5 转账记录

字段：

- 金额
- 转出账户
- 转入账户
- 日期
- 备注

规则：

- 转出账户必填。
- 转入账户必填。
- 转出账户和转入账户不能相同。
- 转账不计入收入或支出统计，只影响账户余额。
- 删除或编辑后，余额自动修正。

### 7.6 金额输入

当前金额键盘支持：

- 数字输入
- 小数点
- 退格
- `+` 和 `-` 连续计算

规则：

- 最多保留 2 位小数。
- 输入总长度上限为 11 个字符。
- 表达式结果必须大于 0 才可保存。
- 例如 `10+2.50` 可以直接计算为 `12.50`。

### 7.7 账目详情

账目详情页当前展示：

- 记录类型/分类信息
- 金额
- 账户信息
- 账目日期
- 备注

并提供：

- 编辑记录
- 删除记录

当前详情页不展示创建时间和更新时间。

### 7.8 定时记账

定时记账用于重复发生的收入、支出和转账。

字段：

- 收支类型
- 重复周期（周、月、年）
- 金额
- 分类
- 账户
- 转出账户
- 转入账户
- 生成时间
- 备注
- 是否启用

规则：

- 定时记账支持收入、支出和转账。
- 到达生成时间后，应用本地生成对应账目，并推进到下一次时间。
- 生成时间固定为 09:00，不提供自定义小时。
- 按周、按月、按年均支持多选配置。
- 收入和支出的账户可不选，分类必填。
- 转账必须选择转出账户和转入账户，且两个账户不能相同。
- 分类或账户不可用导致无法生成时，该定时记账会停用，避免反复失败。

## 12. 数据保存与迁移

### 12.1 保存原则

- 基础账本数据只保存在本地设备。
- 基础记账能力不依赖网络和账号系统。
- 未启用在线助手时，不上传任何账本数据。
- 启用在线助手后，只上传本次问题需要的最小必要数据。
- 当前只支持人民币记账。

### 12.2 导出

当前支持导出单个 JSON 备份文件，备份内容包括：

- 分类
- 账户
- 记录
- 备份版本
- 导出时间
- App 版本号
- 余额符号规则

### 12.3 导入

当前只支持合并导入。

导入规则：

- 记录直接追加，不做去重。
- 分类按“方向 + 名称”复用已有项，不重复插入。
- 账户始终新增；如名称冲突，会自动重命名为“原名称（导入2）”等形式。
- 支持读取备份版本 1 和版本 2。
- 导入旧备份时，会把负债账户余额转换为当前负数规则。

### 12.4 Excel 添加账目

Excel 文件字段包括：

- 日期
- 收支类型
- 分类
- 账户
- 金额
- 备注

规则：

- 当前支持 `.xlsx` 文件。
- 收支类型支持收入和支出，不支持转账。
- 分类按当前已启用分类名称匹配；匹配不到时归到“其他”。
- 匹配不到分类时，备注追加“原分类：原名称”。
- 账户按当前已启用账户名称匹配；匹配不到时不选账户，避免整行失败。
- 空行会跳过；日期、类型、金额错误的行会跳过，其余可正常添加。

### 12.5 换机迁移

用户可在旧设备导出，在新设备导入，恢复分类、账户和记录数据。

### 12.6 在线助手数据边界

默认允许上传：

- 用户本次输入内容。
- 当前可用分类列表。
- 当前可用账户列表。
- 本次问题需要的候选账目。
- 本次问题需要的账单明细，范围可为单日、单周、单月、单年、最近一段时间或全部账单。
- 本次问题需要的统计摘要。
- 用户确认或取消的操作结果元信息。

默认不上传：

- 与本次问题无关的账单明细。
- 完整备份文件。
- 与本次问题无关的历史明细。
- 用户自定义 API Key；该 Key 只在调用用户配置的模型服务时使用，不上传到 Keacs 后端。

后端日志只保留元信息，包括：

- 请求时间。
- 请求耗时。
- 请求状态。
- 错误类型。
- 模型供应商。
- 不含账本内容的操作类型。
- 设备限流标识的不可逆摘要。

后端不保存完整会话内容，不保存完整账本内容。

## 13. 核心业务规则

### 13.1 余额规则

- 资产账户以正数表示。
- 负债账户以负数表示。
- 收入记入账户时加余额。
- 支出记入账户时减余额。
- 转账按“转出减、转入加”处理。
- 账户性质不会改变收支方向，只影响余额符号展示。

### 13.2 历史一致性

- 新增、编辑、删除记录后，账户余额必须同步修正。
- 停用的账户和分类，其历史记录仍要可读。
- 统计结果始终以当前有效记录为准。

### 13.3 Agent 助手写入规则

- Agent 助手不能绕过现有记账规则。
- Agent 助手不能直接写入数据库。
- 所有新增、修改、删除、批量处理和定时记账操作，必须先展示预览。
- 用户确认后，应用按现有业务规则执行写入。
- 用户取消后，不写入任何数据。
- 写入成功后，账户余额和统计必须同步更新。
- 本地业务校验失败时，不执行写入，并提示失败原因。
